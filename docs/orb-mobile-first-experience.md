# ORB Residential — Mobile-First Experience

## Core principle

ORB Residential is expected to be used:

- during shifts
- after incidents
- while emotionally exhausted
- during safeguarding uncertainty
- during handovers
- in low-light environments
- on mobile devices

Therefore:

# mobile is the primary experience.

Desktop is secondary.

---

# Product identity

# ORB Residential

Powered by IndiCare Intelligence.

A standalone premium intelligence companion for adults working in residential care.

---

# UX goals

ORB should feel:

- calm
- fast
- supportive
- emotionally intelligent
- low-pressure
- easy to navigate
- safe during stress

ORB should NOT feel:

- admin-heavy
- enterprise cluttered
- cognitively overwhelming
- dashboard obsessed

# Mobile UX rules

## Large touch targets

Buttons and workflow cards should be usable:

- one-handed
- while walking
- while distracted
- during emotional pressure

---

## Fast startup

Target:

# under 2 seconds to usable ORB.

---

## Minimal navigation depth

Users should reach core workflows in:

# 1–2 taps maximum.

---

## Calm visual hierarchy

Use:

- soft gradients
- dark mode optimisation
- low visual noise
- strong spacing
- conversational layouts
- focused workflows

Avoid:

- crowded dashboards
- complex sidebars
- enterprise table layouts
- excessive widgets

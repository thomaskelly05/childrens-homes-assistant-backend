# ORB Residential — Runtime Productisation Roadmap

## Objective

Move ORB Residential from:

- intelligent prototype
- disconnected assistant systems
- parallel ORB logic

into:

# a converged production-grade residential intelligence platform.

---

# Strategic architecture

## One brain

# IndiCare Intelligence Spine

Shared by:

- ORB Residential
- IndiCare OS
- future mobile apps
- future APIs

---

# Two surfaces

## ORB Residential

Safe standalone intelligence.

Uses:

- user supplied notes
- uploaded files
- saved projects
- memory/preferences
- reflective workflows

Never uses:

- live records
- chronology
- provider dashboards
- safeguarding state
- operational evidence graphs

---

## IndiCare OS

Operational intelligence platform.

Uses:

- chronology
- incidents
- plans
- provider intelligence
- safeguarding state
- oversight systems
- governance systems

# Runtime governance rules

## ORB Residential must never:

- fabricate chronology access
- fabricate provider oversight
- fabricate safeguarding state
- fabricate inspection evidence
- fabricate actions

## ORB Residential must always:

- explain evidence limitations
- encourage human review
- encourage safeguarding escalation where needed
- separate fact from interpretation
- maintain child-centred language

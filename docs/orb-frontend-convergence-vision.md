# ORB Residential — Frontend Convergence Vision

## Objective

Create a calm, premium, emotionally intelligent standalone application experience for:

# ORB Residential

Powered by IndiCare Intelligence.

The frontend should feel:

- premium
- emotionally safe
- mobile-first
- supportive
- focused
- intelligent

NOT:

- enterprise-heavy
- dashboard overloaded
- operational
- compliance obsessed

---

# Core product philosophy

ORB Residential is:

# a companion.

NOT:

# an admin system.

Every frontend decision should reinforce this.

---

# Primary frontend experience

## Opening ORB

Users should feel:

# “I can think here.”

The UI should immediately reduce pressure.

---

# Suggested application structure

## Screen 1 — Welcome / Home

### Elements

- ORB greeting
- quick prompt box
- voice input button
- recent work
- workflow shortcuts
- calming visual hierarchy

### Suggested workflows

- Ask ORB
- Shift Builder
- Record This Properly
- Safeguarding Thinking
- Therapeutic Reframe

---

## Screen 2 — Workflow Workspace

### Purpose

Focused task completion.

### Behaviour

- distraction-free
- conversational
- structured outputs
- workflow-aware rendering
- calm transitions

### Important rule

Never overload users with:

- excessive menus
- side panels
- operational complexity

---

## Screen 3 — Saved Work

### Purpose

Continuity and reflection.

### Includes

- saved outputs
- projects
- pinned reflections
- recent workflows
- favourite prompts

---

## Screen 4 — Profile & Preferences

### Purpose

Shape ORB around the user.

### Includes

- role
- work environment
- support style
- tone preferences
- onboarding progress
- subscription status

---

# Navigation principles

## Maximum simplicity

Primary navigation should remain:

# under 5 main actions.

Avoid:

- enterprise sidebars
- nested admin menus
- operational trees
- dashboard overload

---

# Visual language

## Recommended feel

- soft gradients
- glowing accents
- warm dark mode
- spacious layouts
- conversational spacing
- subtle motion
- emotionally calm typography

---

# ORB presence

ORB should feel:

- present when needed
- quiet when not needed
- intelligent but non-dominating

The interface should never feel:

# noisy.

---

- emotional clarity
- purpose
- support
- guidance

# Runtime rendering

Responses should support:

- structured sections
- collapsible guidance
- evidence indicators
- calm spacing
- workflow-aware formatting
- mobile readability

# Important frontend rule

ORB Residential must NEVER expose:

- chronology dashboards
- provider intelligence
- operational oversight
- workforce management
- governance tooling
- enterprise admin panels

Those belong to:

# IndiCare OS only.

# Mobile-first implementation priorities

## Highest priority

- fast startup
- one-handed use
- quick voice capture
- quick workflow switching
- low cognitive load
- offline-safe drafting later

# Sprint L Full Platform Convergence Audit

Sprint L is a convergence sprint. The repo already contains operational cognition, chronology, governance, workforce, relationship, child impact and therapeutic intelligence. The work is to reveal and route those capabilities through one operating system instead of adding another assistant, dashboard or scoring engine.

## Executive convergence map

| Layer | Canonical path | Sprint L position |
|---|---|---|
| Write plane | Domain service -> metadata extraction -> `ChronologyWriter` -> lifecycle/audit/operational memory | Chronology is the operational truth layer. |
| Read plane | `ChronologyProjectionService`, projection snapshots, governance/workforce OS APIs | Projection-first reads should replace direct fan-out readers. |
| Cognition plane | `build_orb_context()` -> ORB cognition/trajectory/atmosphere/RM/care/regulatory/therapeutic services -> `build_orb_response()` | ORB is operational cognition, not a separate assistant. |
| Surface plane | Care Hub, Child Journey, Chronology, Governance, Workforce, Reports, ORB | Surfaces should consume existing cognition blocks and domain summaries. |
| Compatibility plane | legacy assistant routes, legacy HTML shell, `/os/chronology`, young-people compat routes | Keep for compatibility; do not extend as new OS surfaces. |

## Surfaced in this Sprint L change

- Child journey now calls the existing `child_experience_intelligence_routes.py` endpoint and surfaces status, risk trajectory, emotional distress, child voice visibility, stabilising anchors, relationship markers and inspection questions.
- ORB cognition panels now display `therapeutic_reasoning`, including therapeutic observations, language guardrails and reflective prompt already returned by the backend.
- Care Hub now explicitly answers "What may Ofsted ask about?" and "What remains unresolved?" from existing governance, ORB governance summary, evidence matrix, alerts and actions.
- Child journey ORB links now use `/orb?scope=child&young_person_id=...` instead of assistant compatibility redirects.
- ORB navigation active roots now point at the canonical ORB route only, reducing assistant/voice/settings nav drift.

## Remaining hidden gaps

- `narrative_continuity_service.py` still needs a thin authenticated API if Child Journey is to consume support effectiveness and relationship continuity directly.
- Next chronology still relies heavily on `/os/chronology`; the projection endpoint should become the adapter source before removing compatibility readers.
- Governance and Workforce should prefer ORB RM reflection prompts when a safe context-preview endpoint exists.
- Chronology projections should carry metadata clusters and standards links so reports and governance do not re-derive them.
- Operational event bus adoption is incomplete; write paths still rely on sync hooks in several domains.
- Legacy assistant and HTML shell surfaces remain reachable for compatibility and need measured retirement, not abrupt removal.

## Next recommended sprint

Make chronology projection the shared read adapter. Start with `frontend-next/lib/os-api/chronology.ts`, then ORB context chronology retrieval, then Care Hub chronology-derived widgets. This is the highest-leverage convergence because it reduces duplicate DB fan-out, aligns ORB/Governance/Care Hub with the chronology truth plane, and gives one place to attach metadata clusters, relationship continuity, support effectiveness and standards evidence.
